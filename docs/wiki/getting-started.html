Setup guide

Getting Started

This page turns the repository setup into a predictable checklist. The renderer expects a Vulkan 1.3 capable GPU, a C++20 compiler, CMake, vcpkg, compiled SPIR-V shaders, and runtime assets beside the executable.

Requirements

Software and hardware

Operating system

Windows or Linux. The code has platform definitions for Win32 and Xlib targets.

Compiler

Any C++20 capable compiler supported by the selected CMake generator.

Graphics stack

Vulkan 1.3 driver, Vulkan loader, and validation layers for development.

Dependencies

vcpkg manifest mode installs vk-bootstrap, GLFW, GLM, VMA, stb, tinyobjloader, ImGui, shaderc, and spdlog.

Windows

Configure, build, run

git clone https://github.com/Raiju-Deeq/FYP-Vulkan-Renderer.git
cd FYP-Vulkan-Renderer

git clone https://github.com/microsoft/vcpkg "$env:USERPROFILE\vcpkg"
& "$env:USERPROFILE\vcpkg\bootstrap-vcpkg.bat"

$env:VCPKG_ROOT = "$env:USERPROFILE\vcpkg"
$env:PATH = "$env:VCPKG_ROOT;$env:PATH"

cmake --preset uni-debug
cmake --build --preset uni-debug
.\build\uni-debug\Debug\vulkan-renderer.exe

The executable should be run from, or with access to, the generated output directory because CMake copies assets/ and shaders/ beside the executable after build.

Linux

Configure, build, run

git clone https://github.com/Raiju-Deeq/FYP-Vulkan-Renderer.git
cd FYP-Vulkan-Renderer

git clone https://github.com/microsoft/vcpkg "$HOME/vcpkg"
"$HOME/vcpkg/bootstrap-vcpkg.sh"
export VCPKG_ROOT="$HOME/vcpkg"

cmake --preset linux-debug
cmake --build --preset linux-debug
VK_INSTANCE_LAYERS=VK_LAYER_KHRONOS_validation ./build/linux-debug/vulkan-renderer

On Linux, the CMake file deliberately prefers the system Vulkan loader instead of the vcpkg loader stub so the runtime can find installed ICD driver files.

First checks

Expected runtime behaviour

Window

The app opens a 1280x720 GLFW window titled FYP Vulkan Renderer.

Scene

The default viking_room.obj and viking_room.png assets should render as a rotating textured model.

Debug panel

The ImGui panel shows FPS, frame time, camera distance, frame-time graph, culling, wireframe, normals view, and asset browser buttons.

Resize

Resizing or minimising should not crash. A resize triggers swapchain, per-image sync, pipelines, material descriptors, and ImGui backend recreation.
