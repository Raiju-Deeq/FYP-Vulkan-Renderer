Support guide

Troubleshooting

Most failures in this project fall into five groups: missing Vulkan runtime support, missing dependencies, wrong working directory, stale shader bytecode, or swapchain state changes after resize/minimise.

Build issues

CMake and dependency checks

Symptom Likely cause Fix
CMake cannot find packages. VCPKG_ROOT is not set or vcpkg was not bootstrapped. Set VCPKG_ROOT, put vcpkg on PATH, then re-run the preset.
glslc not found warning. Vulkan SDK or shaderc tools are not available to CMake. Install Vulkan SDK or ensure the vcpkg shaderc tool path exists. Shaders can also be compiled manually.
Linux instance/window extension errors. The app is using the wrong Vulkan loader or no display server compatible with GLFW/Vulkan is available. Install the system Vulkan loader and ICD packages. The CMake file tries to prefer the system loader on Linux.
MSVC warning-as-error build failure. The project uses /W4 /WX; new warnings are treated as blocking defects. Fix the warning rather than disabling broad warning coverage.
Runtime issues

Renderer startup failures

No suitable GPU

VulkanContext requires Vulkan 1.3, Dynamic Rendering, synchronization2, fillModeNonSolid, and bufferDeviceAddress. Update drivers or use a compatible GPU.

Shader file missing

The app opens shaders/mesh.vert.spv and shaders/mesh.frag.spv relative to the working directory. Run from the output directory or ensure CMake copied shaders there.

Asset file missing

The defaults are assets/models/viking_room.obj and assets/textures/viking_room.png. CMake should copy assets beside the executable after build.

Texture load failure

stb_image handles common formats, with ASCII PPM fallback. Check file path, extension, and file integrity.

Rendering issues

Visual output checks

Symptom Likely cause What to inspect
Black window. Shader path, material descriptor, texture upload, or mesh upload failed before draw. Application log, Pipeline::loadSpv, Material::init, Mesh::upload.
Model appears inside-out or invisible with culling. OBJ winding or import transform does not match expected front face. Disable back-face culling in ImGui, enable wireframe, inspect Pipeline front face and loadObjMesh import transform.
Depth looks wrong. Depth attachment, projection matrix, or GLM depth configuration issue. GLM_FORCE_DEPTH_ZERO_TO_ONE, SwapChain::createDepthResources, and pipeline depth state.
Normal debug colours look incorrect. Missing normals in OBJ or bad normal transform. loadObjMesh normal fallback and normals view in mesh.frag.
