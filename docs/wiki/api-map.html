API Map

This page gives you a curated index into the main engine classes, data structures, and source files. Use it when you know roughly what subsystem you want and need to jump straight into the relevant code.

Classes

Main runtime classes

Class Use when you need Source
Application Startup order, UI state, render loop coordination, asset browser, resize recovery. application.hpp / application.cpp
AppWindow GLFW window lifecycle and framebuffer size queries. window.hpp / window.cpp
VulkanContext Vulkan instance/device/queue/surface setup and VMA allocator ownership. vulkan_context.hpp / vulkan_context.cpp
SwapChain Swapchain image acquisition, image views, depth attachments, rebuild behaviour. swapchain.hpp / swapchain.cpp
Pipeline Shader modules, pipeline layout, descriptor set layout, Dynamic Rendering pipeline creation. graphics_pipeline.hpp / graphics_pipeline.cpp
Renderer Per-frame command buffers, sync objects, frame recording, submit, present, ImGui backend. frame_data.hpp / frame_data.cpp
Mesh Device-local vertex/index buffer ownership and draw commands. mesh.hpp / mesh.cpp
Material Texture image, sampler, descriptor pool, descriptor set, descriptor binding. texture.hpp / texture.cpp
GpuBuffer Staging upload helpers for buffers and images, plus resource wrappers. gpu_buffer.hpp / gpu_buffer.cpp
Structs and enums

Data passed between systems

Vertex

Interleaved position, normal, and UV data expected by the mesh pipeline. Defined in the mesh module alongside vertex input descriptions.

LoadedMesh

CPU-side vertices and indices returned by OBJ loading before they are uploaded to the GPU.

LoadedTexture

CPU-side RGBA8 texture pixels and metadata produced by the texture loading path.

DrawPushConstants

MVP matrix and debug mode pushed to vertex and fragment stages each draw.

BufferResource

Vulkan buffer plus VMA allocation wrapper, used by GpuBuffer helpers.

ImageResource

Vulkan image plus allocation, view, format, and extent wrapper used for textures and depth.
