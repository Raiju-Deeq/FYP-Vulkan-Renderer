Assets and Shaders

The current renderer draws one textured OBJ mesh. CPU asset data is loaded with tinyobjloader and stb_image, then staged into device-local Vulkan buffers and images before being bound through the pipeline and material descriptor set.

Runtime assets

Default files

Model

assets/models/viking_room.obj is the default startup mesh. The asset browser can select other .obj files.

Texture

assets/textures/viking_room.png is the default albedo texture. The browser allows .png, .jpg, .jpeg, .bmp, .tga, and .ppm.

Build output

CMake copies assets/ beside the executable after build so relative runtime paths resolve from the output directory.

Prototype evidence

Screenshots live in docs/prototype-evidence/ and are used by the product page.

Mesh loading

OBJ to indexed draw

Parse OBJ

loadObjMesh uses tinyobjloader with triangulation enabled so the draw path can stay as a triangle list.

Repack vertex streams

OBJ positions, normals, and UVs are repacked into the renderer's interleaved Vertex struct: position, normal, texCoord.

Correct import orientation

The current import transform rotates the Viking OBJ by -90 degrees around X at load time, keeping the per-frame model matrix focused on placement and animation.

Upload buffers

Mesh::upload sends vertices and indices through GpuBuffer::uploadBuffer into device-local vertex and index buffers.

Texture loading

Image to descriptor set

Load RGBA8

loadRgba8Texture uses stb_image and forces four channels. ASCII PPM is retained as a fallback for generated prototype textures.

Upload image

GpuBuffer::uploadTexture2D stages pixels, creates an R8G8B8A8_SRGB image, copies pixels, transitions to shader-read layout, and creates an image view.

Create sampler and descriptor

Material::init creates a linear sampler, a one-set descriptor pool, allocates a descriptor set, and writes set 0 binding 0 as a combined image sampler.

Bind before draw

Material::bindDescriptorSet binds the descriptor set against the active pipeline layout before push constants and mesh draw.

Shaders

GLSL and SPIR-V

FileRoleKey interface
shaders/mesh.vertMain vertex shader.Reads position, normal, UV. Push constant contains MVP and debug mode. Outputs UV and normal.
shaders/mesh.fragMain fragment shader.Samples set 0 binding 0. Uses debug mode 0 for lit texture, debug mode 1 for normal visualisation.
shaders/triangle.vert/.fragEarlier prototype shaders.Kept for milestone history and shader compilation coverage.
*.spvCompiled shader bytecode.Generated by glslc through the CMake shader target and copied to the output directory.
