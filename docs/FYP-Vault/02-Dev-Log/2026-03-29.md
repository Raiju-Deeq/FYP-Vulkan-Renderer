## Code Decisions & Architecture Notes

- **`enable_validation_layers(true)` over `request_validation_layers`** - hard-fail on machines without the Khronos validation layer installed. Forces correctness from the start.
- **Surface created before physical device selection** - `vkb::PhysicalDeviceSelector` takes the surface at construction to enforce presentation support.
- **`VkPhysicalDevice` and `VkQueue` not destroyed in `destroy()`** - enumerated handles, not owned allocations. No cleanup required.
