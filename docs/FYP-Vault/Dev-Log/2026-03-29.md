## Code Decisions & Architecture Notes

- **`enable_validation_layers(true)` chosen over `request_validation_layers`**: the project requires validation layers at all times; using the hard-fail variant ensures the build breaks loudly on machines without the Khronos layer installed rather than silently running without validation.
- **Surface created before physical device selection**: `vkb::PhysicalDeviceSelector` takes the surface at construction so it can enforce presentation support on the selected GPU. The creation order instance → surface → physical device is therefore fixed.
- **`VkPhysicalDevice` and `VkQueue` not destroyed in `destroy()`**: these are not owned handles — they are enumerated/retrieved from the driver. Resetting them to `VK_NULL_HANDLE` in `destroy()` is defensive only.
