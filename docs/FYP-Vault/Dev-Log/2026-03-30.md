### Concept Learning — M1 Vulkan Object Chain
- Studied and documented the full Vulkan object chain required for M1 (instance through to triangle), saved to `docs/Learnings/Learnings1.md`.
- Key concepts covered:
  - **VkInstance** — application's connection to the Vulkan loader
  - **VkSurfaceKHR** — bridge between Vulkan and the windowing system
  - **VkPhysicalDevice** — GPU selection with feature requirements (dynamicRendering, synchronization2, bufferDeviceAddress)
  - **VkDevice** — logical device, the factory for all Vulkan objects
  - **VkQueue** — asynchronous GPU work submission
  - **VkSwapchain** — double-buffered image ring for presentation
  - **VkPipeline** — compiled GPU program with Dynamic Rendering (no VkRenderPass)
  - **Command Buffers + Synchronisation** — per-frame record/submit/present with semaphores and fences
  - **Image Layout Transitions** — explicit `VkImageMemoryBarrier2` via synchronization2
