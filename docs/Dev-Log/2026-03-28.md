I rewrote and unified my FYP Contract and Project Definition into a single v9.0 document.

  both the university's template requirements and the formal supervisory contract structure.

  scope and intent.

  Physics, ECS, audio, shadow mapping, skeletal animation, and mobile support are all out
  of scope by design — not by accident.
- **Added a full Risk Register** with 8 risks, likelihood/impact ratings, and concrete
  mitigations including a specific Gaussian splatting scope creep guardrail.

### Session 1 — Morning (2–3 hrs) · `VulkanContext`

This is the most important session of the whole project. Get this right and every
subsequent milestone builds cleanly on top of it.

**Tasks:**
- [ ] Confirm `vk-bootstrap` is in `vcpkg.json` and resolves correctly
- [ ] Write `VulkanContext.h` — declare instance, physical device, logical device,
      graphics/present queues, surface, debug messenger
- [ ] Write `VulkanContext.cpp` — implement `init()` using vk-bootstrap:
  - `vkb::InstanceBuilder` → Vulkan 1.3, validation layers, debug messenger
  - `vkb::PhysicalDeviceSelector` → require Vulkan 1.3 + dynamic rendering feature
  - `vkb::DeviceBuilder` → logical device
  - Extract graphics and present queues via `vkb::Device::get_queue()`
- [ ] Implement `cleanup()` with correct destruction order

**Done when:** Terminal shows validation layer active, context initialises and tears down
with zero errors.
