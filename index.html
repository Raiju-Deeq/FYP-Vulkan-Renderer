What it is

A renderer is the part of software that turns data into images.

Games, simulations, virtual production tools, and creative apps all need rendering. Most engines hide how that process works. Raiju Renderer is different: it is designed to show the pipeline clearly, from loading a 3D model to sending commands to the GPU and presenting the final image on screen.

Current prototype

Small enough to understand. Real enough to matter.

The current branch focuses on a complete vertical slice: open a window, initialise Vulkan, create a swapchain, compile shaders, upload a model and texture, render with depth, show debug controls, and recover cleanly when the window is resized.

  • ✓Loads and renders a textured OBJ model.
  • ✓Uses Vulkan 1.3 Dynamic Rendering instead of the older render pass path.
  • ✓Includes live debug controls for FPS, frame time, wireframe, culling, and normals view.
  • ✓Uses explicit GPU synchronisation and resource ownership rules.

How it works

The render path, in plain English.

Raiju Renderer is not trying to hide the hard parts. It organises them into clear steps so the rendering process can be followed, tested, and extended.

Load the assets

The renderer reads a 3D model and texture from disk, then prepares the data so the GPU can use it efficiently.

Upload to the GPU

Model and texture data are copied into GPU-friendly resources using explicit Vulkan memory and transfer commands.

Record the frame

Each frame records commands that bind the pipeline, bind the texture, draw the mesh, render the debug UI, and prepare the image for presentation.

Present the result

The completed image is handed to the display system, while fences and semaphores keep CPU, GPU, and presentation work in the correct order.

Why open source?

The goal is to make the renderer useful as a public learning resource. People should be able to inspect the code, study the architecture, build it locally, and understand why each system exists.

Roadmap

Where the project goes next.

The renderer is intentionally built in stages. The first priority is correctness and clarity. More advanced features come after the foundation is stable.

Current

Textured Vulkan rendering vertical slice

Window creation, Vulkan setup, swapchain rendering, OBJ loading, texture upload, debug UI, wireframe mode, normals view, depth, and resize recovery.

Next

Better project polish and documentation

Cleaner examples, screenshots, guided build instructions, RenderDoc walkthroughs, and contributor-friendly documentation.

Future

Advanced rendering foundations

Improved materials, scene loading, lighting, performance instrumentation, and research-driven experiments such as Gaussian splatting.

Follow the renderer as it grows.

Raiju Renderer is early, but the direction is clear: a transparent, open-source Vulkan renderer that helps more people understand how real-time graphics work.
